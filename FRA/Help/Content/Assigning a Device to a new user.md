Il peut être nécessaire de reprovisionner un périphérique déjà enregistré : pour un nouvel utilisateur, si le rôle d'un utilisateur a changé ou si la relation du précédent utilisateur avec l'entreprise a évolué. Cette procédure évite d'exclure un périphérique et de le réenregistrer.

**Procédure** :

1. Accédez au périphérique dans la page [**Périphériques**](./Devices-.md).
2. Cliquez sur le nom du périphérique pour afficher la page des détails de ce dernier.
3. Cliquez sur l'icône **Affecter à l'utilisateur**![](Assign_user_icon.png).
4. ****************Sinon, vous pouvez sélectionner un périphérique dans la page Périphériques, puis cliquer sur l'option Affecter à l'utilisateur dans le menu Actions. .
5. Sélectionnez l'utilisateur souhaité.
6. Cliquez sur **Affecter à l'utilisateur**.

    Le périphérique sera provisionné pour cet utilisateur.


 Sachez que dans les scénarios basés sur les utilisateurs et les licences, vous pouvez attribuer un périphérique à un utilisateur qui a dépassé la limite de périphériques définie. La limite de périphériques a pour objectif de restreindre l'inscription de périphériques dans les scénarios BYOD (Bring Your Own Device, prenez vos périphériques personnels).

Il est sans conséquence de dépasser la limite de périphériques pour les licences basées sur les périphériques et sur les utilisateurs. Dans le cas des licences basées sur les périphériques, le coût pour le client final ne varie pas étant donné que le nombre total de périphériques dans le système ne change pas. Dans le cas des licences basées sur les utilisateurs, l'absence de vérification profite au client. Par exemple, si l'entreprise compte cinq utilisateurs (U1 à U5) chacun possédant cinq périphériques, avec une licence basée sur les utilisateurs, cela signifie qu'elle utilise cinq licences. Si deux des périphériques des utilisateurs U4 et U5 sont transférés aux utilisateurs U1 et U2, la consommation de licences DIMINUE et passe de cinq à trois.
