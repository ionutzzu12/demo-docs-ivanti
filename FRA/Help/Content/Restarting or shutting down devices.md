 Cette section traite des sujets suivants :
- Redémarrer un périphérique
- Arrêter un périphérique


**S'applique à :** Android 7.0+ (périphériques gérés), périphériques supervisés iOS 10.3+ (iOS et tvOS), périphériques macOS 10.13+ et Windows 10+.

Les administrateurs peuvent redémarrer ou arrêter un périphérique supervisé iOS ou tvOS individuellement à partir de la page Informations sur le périphérique ou en grande quantité depuis la page Liste des périphériques.

## [****](#)Redémarrer un périphérique

**Procédure**

1. Accédez à **Périphériques**.
2. Repérez le périphérique.
3. Cliquez sur le lien dans la colonne **Nom**.
4. Cliquez sur le bouton **Actions**.
5. Cliquez sur **Redémarrer/Éteindre l'appareil**.
 Les périphériques non pris en charge ne peuvent pas être redémarrés.
6. Lisez l'avertissement qui s'affiche.
7. (Facultatif) Sélectionnez l'option permettant d'effacer le code d'accès de l'appareil au redémarrage. Si le code d'accès n'est pas effacé, l'appareil exigera un code d'accès et ne pourra pas se connecter au Wi-Fi après le redémarrage.
8. Sélectionnez **Redémarrer le périphérique** si ce n'est pas déjà le cas.
9. Si vous souhaitez toujours redémarrer l'appareil, cliquez sur **Envoyer à l'appareil**. Sinon, cliquez sur **Annuler**.
 Pour les appareils Android, les administrateurs peuvent consulter les informations sur la date et l'heure du redémarrage de l'appareil sous **Temps de fonctionnement** dans la page Détails de l'appareil.

## [****](#)Arrêter un périphérique

**Procédure**

1. Accédez à **Périphériques**.
2. Repérez le périphérique.
3. Cliquez sur le lien dans la colonne **Nom**.
4. Cliquez sur le bouton **Actions**.
5. Cliquez sur **Redémarrer/Éteindre l'appareil**.
 Les périphériques non pris en charge ne peuvent pas être redémarrés.
6. Lisez l'avertissement qui s'affiche.
7. Sélectionnez **Arrêter le périphérique**.
8. Si vous souhaitez toujours éteindre l'appareil, cliquez sur **Envoyer à l'appareil**. Sinon, cliquez sur **Annuler**.
