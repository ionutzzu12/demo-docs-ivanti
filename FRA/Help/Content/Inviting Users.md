**Procédure**

1. Accédez à **Utilisateurs**.
2. Sélectionnez les utilisateurs que vous voulez inviter.
3. ****************L'aperçu de l'invitation apparaît, ainsi qu'une option pour définir la propriété du périphérique sur Propriété de l'utilisateur ou Propriété de l'entreprise. 


::Image[]{src="Resources/Images/cloud67regconf001.png" size="100" caption alt isUploading="false" sha initialPath="Resources/Images/cloud67regconf001.png" githubPath="Content/Resources/Images/cloud67regconf001.png" indent="2"}

4. Si vous le souhaitez, activez les **paramètres de propriété du périphérique**.
5. Cliquez sur **Appartenant à l'utilisateur** ou **Appartenant à l'entreprise**. Ce paramètre modifie la classification de l'appareil lors de l'enregistrement. Il s'applique uniquement aux enregistrements par code PIN ou par mot de passe. Si **Paramètres du propriétaire de l'appareil** est désactivé, les appareils seront enregistrés comme « Non défini ». Pour les appareils supervisés, le paramètre du propriétaire sera « Appartenant à l'entreprise ».
6. Cliquez sur **Envoyer**. Si l'enregistrement de l'appareil s'est fait par code PIN, l'utilisateur recevra un code PIN à son adresse e-mail enregistrée. Si l'enregistrement s'est fait par code QR, l'utilisateur recevra un code QR.
7. Cliquez **OK**.
 Si la fonctionnalité d'e-mail de confirmation d'inscription est activée comme décrit dans la section « Configuration et utilisation des e-mails de confirmation d'inscription », un message vous rappellera que l'utilisateur recevra un e-mail de confirmation après son inscription. Pour recevoir cet e-mail, l'utilisateur doit figurer sur la liste de diffusion que vous avez définie dans la section « Configuration des e-mails de confirmation d'inscription » des paramètres utilisateur.


Pour plus d'informations, voir [**Importation des utilisateurs LDAP**](\<./ Configuration du serveur LDAP.md>).
