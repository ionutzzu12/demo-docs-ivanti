 Cette section traite des sujets suivants :
- Visualiser la distribution des applis
- Afficher les détails de l'appli
- Ajout de graphiques de distribution d'une appli unique
- Ajout d'un graphique des applis iOS non gérées
- Ajout des dix applis gérées les plus installées
- Ajout des cinq applis internes les mieux notées


App Insights (Infos d'appli) est une fonction de tableau de bord qui permet d'afficher et d'analyser les distributions d'applis suivantes :
- Distribution d'applis internes nécessitant une installation
- Distribution d'applis publiques nécessitant une installation
- Applis iOS non gérées
- Dix applis gérées les plus installées
- Cinq applis internes les mieux notées


Analyse des 5 applis nécessitant une installation en priorité : il s'agit des applis internes ou publiques distribuées à un grand nombre d'utilisateurs mais présentant proportionnellement un faible taux d'installation. Le graphique des applis iOS non gérées contient des informations sur les applis non gérées installées sur les périphériques. Vous pouvez ainsi consulter la liste des applis non gérées et des périphériques sur lesquels elles sont installées, ainsi que convertir ces applis en applis gérées. Ce type de distribution d'applis demande une attention ou une action particulière de l'administrateur pour améliorer la distribution. Ces graphiques représentent les périphériques sur lesquels les applis sont déjà installées. Le diagramme circulaire fournit un aperçu global de la distribution des applis publiques et internes. Il facilite l'analyse du nombre de périphériques sur lesquels des applis doivent être installées et permet d'obtenir des informations spécifiques sur les différentes applis en cliquant sur la zone du diagramme correspondante. Vous pouvez également ajouter un graphique de distribution unique qui représente la distribution d'une version spécifique d'une appli unique.

Le tableau de bord affiche uniquement les informations sur les périphériques qui ont pris contact au cours des 15 derniers jours.

## [****](#)Visualiser la distribution des applis

Sur la page **Applis** dans le **Tableau de bord**, vous pouvez visualiser les graphiques suivants :
- Distribution d'applis internes nécessitant une installation
- Distribution d'applis publiques nécessitant une installation
- Applis iOS non gérées


Les graphiques affichés par défaut sont : 5 applis internes, 5 applis publiques et Applis iOS non gérées. Les graphiques sont organisés de gauche à droite, en commençant par l'appli qui présente le plus faible taux d'installation.

Les graphiques en anneau utilisent 2 couleurs pour indiquer l'état de l'installation. La couleur bleue représente le nombre de périphériques sur lesquels l'appli est installée. La couleur rouge représente le nombre de périphériques sur lesquels une installation est nécessaire. Pour afficher le nombre de périphériques, passez le pointeur de la souris sur chaque zone. 

Pour supprimer un graphique, cliquez sur l'option Supprimer en haut à droite du graphique.

## [****](#)Afficher les détails de l'appli

Au centre du diagramme circulaire figure le nombre de périphériques sur lesquels une installation est nécessaire. Exemple : 750/1 000 indique que 750 périphériques sur 1 000 nécessitent une installation. 

Les graphiques en anneau utilisent 3 couleurs pour représenter la distribution de l'appli.
- La couleur bleue représente le nombre de périphériques sur lesquels l'appli est installée. Cliquez sur la zone bleue du diagramme pour accéder à la page **Périphériques**. Sur la page **Appareils** , la colonne **Version de l'application** affiche la version de l'application installée et la date d'installation.
 Vous pouvez également afficher les appareils en fonction des versions des applications installées en sélectionnant les options dans la section **Versions** du panneau de gauche.
- La couleur rouge représente le nombre de périphériques sur lesquels une installation de l'appli est nécessaire. Cliquez sur la zone rouge du diagramme pour accéder à la page **Périphériques**. Sur la page **Périphériques**, vous pouvez voir les périphériques sur lesquels l'appli doit être installée.


L'icône de l'appli s'affiche au centre du diagramme. Cliquez sur l'icône pour accéder à la page des détails de l'appli sous **Applis > Catalogue d'applis**.

Sur cette page, sélectionnez l'onglet **Périphériques avec l'appli installée** pour afficher la liste des périphériques sur lesquels l'appli sélectionnée est installée. 

Pour afficher la liste des périphériques sur lesquels l'appli sélectionnée n'est pas installée, sélectionnez l'onglet **Périphériques sans l'appli installée**.

## [****](#)Ajout de graphiques de distribution d'une appli unique

Sur la page Applis, vous pouvez ajouter un graphique de distribution en anneau unique pour une version spécifique de l'appli. La couleur rouge représente la liste des périphériques compatibles sur lesquels l'appli est normalement installée. Ces graphiques fournissent une représentation visuelle des informations suivantes sur la distribution des applis :
- Périphériques sur lesquels une version spécifique de l'appli est installée
- Périphériques sur lesquels une autre version de l'appli est installée
- Périphériques sur lesquels l'appli n'est pas installée


**Procédure**

1. Cliquez sur **+ Ajouter** sur la page **Applis**. La fenêtre **Ajouter un graphique d'appli** s'affiche.
2. Dans la liste déroulante **Type de graphique**, sélectionnez **Distribution d'une appli unique**.
3. Cochez la case correspondant à la liste des applications pour lesquelles vous souhaitez afficher le graphique de distribution individuel.

 Vous pouvez également rechercher une application spécifique en saisissant son nom dans le champ « Rechercher des applications ».
4. Cliquez sur **Ajouter un graphique**. Les graphiques de répartition des applications individuelles sont affichés sur la page Applications.

 Vous pouvez sélectionner un maximum de 9 applications dans la liste.


Au centre du graphique figure le nombre de périphériques sur lesquels la version spécifique de l'appli est installée.

Exemple : 5/10 indique que 5 périphériques sur 10 disposent de la version spécifique de l'appli.

Les graphiques en anneau utilisent 3 couleurs pour représenter la distribution de l'appli.
- Le vert représente le nombre de périphériques sur lesquels une version spécifique de l'appli est installée. Cliquez sur la zone verte du diagramme pour accéder à la page **Périphériques** où figure la liste des périphériques sur lesquels la version spécifique de l'appli est installée. Vous pouvez également afficher les périphériques disposant d'une autre version de l'appli en sélectionnant les versions souhaitées dans la section **Versions**, dans le volet de gauche.
- Le vert clair représente le nombre de périphériques sur lesquels d'autres versions de l'appli sont installées. Cliquez sur la zone vert clair du diagramme pour accéder à la page **Périphériques** où figure la liste des périphériques sur lesquels une autre version de l'appli est installée. Vous pouvez également afficher les périphériques disposant d'une autre version de l'appli en sélectionnant les versions souhaitées dans la section **Versions**, dans le volet de gauche.
- La couleur rouge représente le nombre de périphériques sur lesquels l'appli n'est pas installée. Pour afficher le nombre de périphériques, passez le pointeur de la souris sur chaque zone. Cliquez sur la zone rouge du diagramme pour accéder à la page **Périphériques** où figurent les périphériques sur lesquels l'appli n'est pas installée. Le volet de gauche affiche également la date de mise à disposition de l'appli dans le catalogue d'applis.


L'icône de l'appli s'affiche au centre du diagramme. Cliquez sur l'icône pour accéder à la page des détails de l'appli sous **Applis > Catalogue d'applis**. Sur cette page, sélectionnez l'onglet **Périphériques avec l'appli installée** pour afficher la liste des périphériques sur lesquels l'appli sélectionnée est installée. Pour afficher la liste des périphériques sur lesquels l'appli sélectionnée n'est pas installée, sélectionnez l'onglet **Périphériques sans l'appli installée**.

Pour supprimer un graphique, cliquez sur l'option Supprimer en haut à droite du graphique.

## [****](#)Ajout d'un graphique des applis iOS non gérées

Vous pouvez identifier les applis non gérées et en afficher la liste en ajoutant le graphique des applis iOS non gérées sur la page Applis. Ce graphique s'affiche automatiquement lorsqu'un administrateur ajoute une appli iOS non gérée au catalogue. L'administrateur peut supprimer ou ajouter ce graphique dès que nécessaire.

**Procédure**

1. Cliquez sur **+ Ajouter** sur la page **Applis**. La fenêtre **Ajouter un graphique d'appli** s'affiche.
2. Dans la liste déroulante **Type de graphique**, sélectionnez **Applis iOS non gérées**.
3. Cliquez sur **Ajouter un graphique**. Le diagramme des applis iOS non gérées apparaît sur la page **Applis**.


Ce graphique indique le nombre d'applis non gérées dans le catalogue d'applis. Au bas du graphique, trois colonnes comprennent les détails suivants :
- **Devices with unmanaged iOS apps** : indique le nombre d'applis iOS non gérées. Cliquez sur le lien pour afficher la liste des périphériques contenant des applis non gérées dans la fenêtre « Devices with unmanaged iOS apps »..
- **Nbre total d'applis dans le catalogue d'applis** : Indique le nombre total d'applis disponibles dans le catalogue d'applis.
- **Unmanaged iOS apps (%)** : indique le pourcentage d'applis iOS non gérées.


Si une appli est déjà installée à partir de l'App Store iTune, vous pouvez convertir l'appli et ses données en appli gérée. Pour convertir ce genre d'applis en appli gérée :

1. Cliquez sur le lien du nombre indiqué dans la colonne **Devices with unmanaged iOS apps**. La fenêtre **Applis iOS non gérées** apparaît.
2. Sélectionnez une ou plusieurs applis non gérées dans la liste, puis cliquez sur le lien de nombre sous les applis iOS non gérées. Les applications sélectionnées seront converties en applications gérées et leur statut sera mis à jour lors du prochain enregistrement de l'appareil.
 Vous pouvez exporter les données des applications non gérées au format CSV en cliquant sur le lien **Exporter au format CSV** .


## [****](#)Ajout des dix applis gérées les plus installées

Vous pouvez identifier et afficher la liste des dix applis gérées les plus installées dans un graphique sur la page Applis. L'administrateur peut supprimer ou ajouter ce diagramme au besoin, à tout moment.

Par défaut, le graphique des dix applis gérées les plus installées est disponible sur la page **Applis**. Si le graphique est supprimé, l'administrateur peut l'ajouter depuis la page **Applis**.

**Procédure**

1. Cliquez sur **+ Ajouter** sur la page **Applis**. La fenêtre **Ajouter un graphique d'appli** s'affiche.
2. Dans la liste déroulante **Type de graphique**, sélectionnez **Dix applis gérées les plus installées**.
3. Cliquez sur **Ajouter un graphique**. Le diagramme des dix applis gérées les plus installées apparaît sur la page **Applis**.


Vous pouvez afficher les dix applis gérées les plus installées en fonction de la catégorie sélectionnée dans la liste déroulante **Afficher**. Les catégories disponibles sont :
- **Toutes les applis** (sélectionnée par défaut)
- **Applis internes**
- **Applis publiques**


Chaque barre du diagramme représente une appli spécifique dont le nom est indiqué. Survolez chaque barre pour connaître la plateforme (Android, iOS ou Windows) et le nombre de périphériques sur lesquels l'appli est installée. 

Cliquez sur la barre d'une appli pour accéder à la page **Périphériques** contenant les informations relatives aux périphériques sur lesquels l'appli est installée. Sur cette page, le volet de gauche comprend le nombre de périphériques sur lesquels l'appli est installée. Cliquez sur la croix (X) dans le volet de gauche pour revenir à la page **Applis** du tableau de bord. 

Vous pouvez supprimer le graphique en cliquant sur l'option de suppression, en haut à droite du graphique.

## [****](#)Ajout des cinq applis internes les mieux notées

Vous pouvez identifier et afficher la liste des cinq applis internes les mieux notées dans un graphique sur la page Applis. L'administrateur peut supprimer ou ajouter ce diagramme au besoin, à tout moment.

Par défaut, le graphique des cinq applis internes les mieux notées est disponible sur la page **Applis**. Si le graphique est supprimé, l'administrateur peut l'ajouter depuis la page **Applis**.

**Procédure**

1. Cliquez sur **+ Ajouter** sur la page **Applis**. La fenêtre **Ajouter un graphique d'appli** s'affiche.
2. Dans la liste déroulante **Type de graphique**, sélectionnez **Cinq applis internes les mieux notées**.
3. Cliquez sur **Ajouter un graphique**. Le diagramme des cinq applis internes les mieux notées apparaît sur la page **Applis**.


Ce graphique représente les applis par leur logo, ainsi que la note qui leur est attribuée par des images d'étoiles et le chiffre associé (la note maximale étant 5). Le nombre d'utilisateurs ayant accordé une note à l'appli est également indiqué.

Les évaluations d'une appli ne proviennent pas uniquement des périphériques gérés par cet administrateur dans cet espace, mais de tous les utilisateurs de l'appli. La note finale d'une appli correspond à la moyenne de toutes les notes accordées par différents utilisateurs ayant affiché cette appli dans Apps@Work sur leurs périphériques enregistrés.

Cliquer sur l'appli affiche la page **Informations sur l'appli**, qui contient des détails spécifiques à cette appli.

Vous pouvez supprimer le graphique en cliquant sur l'option de suppression, en haut à droite du graphique.
