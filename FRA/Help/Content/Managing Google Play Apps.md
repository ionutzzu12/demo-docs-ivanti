Vous pouvez définir le fichier binaire de l'application Google Play à déployer auprès de groupes ou d'utilisateurs spécifiques. Ce déploiement s'applique aux déploiements Android en entreprise. Le développeur de l'application doit également ajouter votre organisation à la liste blanche pour pouvoir déployer des applications des canaux alpha ou bêta.

1. Cliquez sur **Applis**.
2. Dans le **catalogue d'applis**, sélectionnez une appli pour laquelle paramétrer la configuration Version de Google Play.
3. Cliquez sur l'onglet **Configurations des applis**.
4. Cliquez sur **Version de Google Play**.
5. La configuration de publication Google s'applique uniquement aux applications Android d'entreprise. Par défaut, l'option Production est utilisée si la configuration de publication Google n'est pas sélectionnée pour les nouvelles applications.
 Cliquez sur **Ajouter**.
6. Donnez un nom pour la configuration.
7. Si vous le souhaitez, ajoutez une description.
8. Sélectionnez une option dans la liste déroulante pour sélectionner le fichier binaire auquel les utilisateurs et périphériques destinataires de l'appli auront accès. 
   - Production
   - Alpha
   -  L'option Production est appliquée par défaut pour les applis qui sont déjà déployées sur le périphérique.

9. Configurez les options de distribution en sélectionnant **Toutes les personnes dotées de l'appli**, **Personne** ou **Personnalisé**.
10. ********L'option Personnalisé distribue l'appli au groupe d'utilisateur, selon le filtre de périphérique appliqué.


## Classer plusieurs configurations Version de Google Play par ordre de priorité

Lorsque vous ajoutez plusieurs configurations Version de Google Play, vous pouvez définir dans quel ordre elles doivent être appliquées.

1. Dans **Configurations des applis**, cliquez sur** Donner la priorité à certaines configurations**.
2. Ce bouton s'affiche uniquement lorsque plusieurs configurations sont listées.

 Parmi les configurations listées, faites glisser et déposez en haut de la liste celle qui doit être appliquée en priorité.
3. **** Quand la configuration prioritaire est supprimée, la configuration qui était deuxième dans l'ordre de priorité la remplace.
