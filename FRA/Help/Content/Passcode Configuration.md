L'une des premières choses que vous configurez (à l'aide de l'assistant de démarrage) est la configuration du code d'accès. Cette configuration définit les paramètres de la fonction de verrouillage d'écran sur les appareils.

## Configuration du code d'accès

|  **Paramètre**                                                 |  **Action**                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
|----------------------------------------------------------------|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
|Nom|Saisissez un nom qui identifie cette configuration.|
|Description|Saisissez une description qui précise l'objectif de cette configuration.|
|Autoriser les valeurs simples|Définit si le PIN ou le mot de passe peut contenir des caractères alphanumériques triés.<br /> <br />****Pour iOS, macOS, watchOS et Android : Sélectionnez cette option pour autoriser les codes PIN ou codes d'accès moins sécurisés parce qu'ils contiennent des caractères répétés, ou des séries de caractères croissantes ou décroissantes.<br />Exemples : 1111, 1234, abcd.<br /> <br />Désactiver cette option sur les appareils Android imposera des codes d'accès complexes. Par exemple, les utilisateurs ne pourront pas configurer de séquences de caractères répétées, croissantes ou décroissantes.<br />**Pour Windows 10+ :** Sélectionnez cette option pour autoriser les codes d'accès qui sont moins sûrs parce qu'ils contiennent des suites de chiffres répétés croissantes ou décroissantes.<br />Exemples : 1111, 1234.|
|Exiger une valeur alphanumérique|Oblige à utiliser au moins une lettre et un chiffre dans le code d'accès.<br /> <br />****Pour iOS et Android : Sélectionnez cette option pour exiger l'utilisation de codes d'accès contenant des lettres et des chiffres.<br />**Pour Windows 10+ :** Sélectionnez cette option pour exiger l'utilisation de codes d'accès sûrs basés sur la norme Microsoft.|
|Longueur minimum du code d'accès|Sélectionnez un chiffre dans la liste pour définir une longueur minimale pour les codes d'accès.<br />**Pour Windows 10 Desktop** : les comptes locaux devront respecter une longueur minimum du code d'accès de 6 caractères.|
|Nombre minimum de caractères complexes|****Pour iOS et Android : Sélectionnez un chiffre dans la liste pour définir le nombre minimal de caractères autres que les chiffres ou les lettres.<br />**Pour Windows 10+** : Impose 3 caractères complexes pour les comptes locaux.|
|Durée de validité maximale du code d'accès|Saisissez le nombre de jours après lesquels l'utilisateur doit réinitialiser le code d'accès. Si vous ne souhaitez pas définir de durée de validité pour le code d'accès, laissez ce champ vide.|
|Verrouillage automatique|Sélectionnez un intervalle dans la liste pour définir la durée pendant laquelle le périphérique peut rester inactif avant le verrouillage automatique de l'écran.|
|Toute méthode de verrouillage|****|
|Historique du code d'accès|Saisissez un nombre pour définir le nombre de codes d'accès uniques qu'un utilisateur doit saisir avant de pouvoir réutiliser un code d'accès. Par exemple, si vous définissez ce champ sur 4, l'utilisateur devra définir 4 codes d'accès avant de pouvoir réutiliser le premier.|
|Période de tolérance avant le verrouillage du périphérique|Choisissez dans la liste l'intervalle entre l'apparition de l'écran de verrouillage et le moment auquel l'utilisateur du périphérique doit entrer un code d'accès pour déverrouiller le périphérique.|
|Nombre maximum d'échecs|Choisissez dans la liste le nombre de fois où l'utilisateur peut entrer un code d'accès incorrect avant que le périphérique soit réinitialisé et ses données effacées. <br />**Avertissement :** Le contenu des périphériques sera supprimé si l'utilisateur dépasse le nombre maximal de tentatives de saisie infructueuse du mot de passe. |
|(macOS uniquement)<br />Activer l'expression régulière de code d'accès (macOS 14+)|Spécifiez la chaîne d'expression correspondant au mot de passe pour vérifier si elle répond à la stratégie. |
|(macOS uniquement)<br />Langage|<br />Spécifiez la langue de la description.|
|(macOS uniquement)<br />Description|<br />Décrivez la complexité du mot de passe. Par exemple : chiffres, caractères spéciaux, chaînes de caractères, etc.|
|(macOS uniquement)<br />Appliquer la règle relative au code d'accès à la prochaine connexion|Sélectionnez cette option pour que macOS envoie à l'utilisateur, à l'occasion de sa connexion suivante, une demande de modification du mot de passe dans le but de le rendre conforme à la stratégie de mot de passe.<br />Par défaut, cette option est désactivée.<br />Disponible sous macOS 10.13 et versions ultérieures.|
|(macOS uniquement)<br />Minutes avant réinitialisation en cas d'échec de connexion|Spécifiez le nombre de minutes avant réinitialisation du nom de connexion si le nombre maximal d'échecs de connexion a été atteint.<br />Assurez-vous que le nombre maximal de tentatives infructueuses est défini pour activer ce champ. Disponible sous macOS 10.10 et versions ultérieures.|
|Smart Lock |Pour les périphériques Android 5.0, excepté dans les profils professionnels Android Enterprise :<br />**Pour Android 6.0 ou supérieur :**<br />Permet ou non à l'utilisateur d'utiliser la fonction SmartLock pour déverrouiller l'appareil. SmartLock déverrouille automatiquement l'appareil dans certaines circonstances, par exemple lorsque l'utilisateur est à proximité, lorsque l'appareil se trouve à un endroit précis ou lorsqu'il est jumelé avec un appareil de confiance.|
|Déverrouillage à l'aide de l'empreinte digitale |Pour les périphériques Android 5.0, excepté dans les profils professionnels Android Enterprise :<br />**Pour Android 6.0 ou supérieur :**<br />Permet ou non à l'utilisateur de choisir l'utilisation de l'empreinte digitale pour déverrouiller un périphérique.|
|Notifications de l'écran de verrouillage (pour Android Enterprise uniquement)|Activer les notifications pour les périphériques professionnels gérés (propriétaire du périphérique)<br />Autorisez ou non l'affichage de notifications sur l'écran de verrouillage des périphériques gérés par profil professionnel.<br /> <br />Activer les notifications non modifiées pour Work Profile<br />**Pour Android 6.0 ou supérieur :**<br />Autorisez ou non l'affichage de notifications non modifiées sur l'écran de verrouillage des périphériques avec profil professionnel.<br /> Une fois ce paramètre activé, vous recevez les notifications, mais leur contenu est masqué en application des stratégies définies. Vous ne pouvez voir ce contenu (notification push/d'e-mail) qu'à partir de l'appli.|

[****](./Configurations.md)Pour en savoir plus, consultez la rubrique Création d'une configuration.
