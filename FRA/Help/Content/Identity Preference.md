**S'applique à** : macOS 10.12 ou versions plus récentes compatibles.

Identifiez un élément de Préférence d'identité dans le trousseau de l'utilisateur qui référence une charge de traitement Identité incluse dans le même profil.

****

 crée une configuration système Préférence d'identité par défaut avec une charge de traitement relative à l'URL AppStore et le certificat à utiliser.

Lors de l'accès au catalogue d'apps macOS via Safari sous macOS 10.12 et versions ultérieures, une invite de mot de passe système s'affiche afin de mettre en cache le certificat d'identité. Il est nécessaire de sélectionner « Toujours autoriser » lors de la première utilisation pour éviter que cette invite ne s'affiche ultérieurement.

Safari sous une version de macOS antérieure à 10.12 et les autres navigateurs affichent des messages relatifs aux certificats et au mot de passe système lorsque les utilisateurs accèdent au catalogue d'applis macOS dans une nouvelle session du navigateur à partir de périphériques macOS.

## Création d'une configuration de préférence d'identité

**Procédure**

1. Sélectionnez **Configurations**.
2. Cliquez sur **+ Ajouter**.
3. Dans le champ de recherche, saisissez **préférence**, puis cliquez sur la configuration **Préférence d'identité**.
4. Nommez et décrivez la configuration.
5. Dans la section Installation de la configuration, dans le champ **Nom**, saisissez un identifiant d'e-mail, un nom d'hôte DNS ou un nom identifiant de façon unique le service.
6. Dans le champ **UUID du certificat**, sélectionnez un certificat.
7. Cliquez sur **Suivant** pour configurer les paramètres de distribution.
8. Cliquez sur **Terminé**.
