 L'onglet Apps@Work offre les fonctions suivantes :
-  Accéder à l'onglet Apps@Work depuis l'application 
- Rechercher
- Installation d'une appli - États du bouton
- Applications présentées et bannière
- Notification de mise à jour de l'application
- Paramètres - Mes périphériques


### [****](#) Accéder à l'onglet Apps@Work depuis l'application 

**Procédure**

1.  Connectez-vous à  avec votre périphérique iOS. 
2. Appuyez sur l'icône **Applications\\@Travail** . Deux onglets par défaut sont disponibles : Toutes les applications et Catégories.

   ![](<Resources/Images/Toutes les applications.png>)
3. ****L'onglet Toutes les applis répertorie toutes les applis dans l'ordre alphabétique.
4. ****L'onglet Catégories affiche uniquement les catégories comportant au moins une application, comme suit : 
   - L'écran indique pour chaque catégorie le nombre d'applications présentes.
   - La ligne « Mes applications » de l’onglet « Catégories » est une liste contenant toutes les applications installées. « Mes applications » est toujours la première catégorie ; les autres catégories sont classées par ordre alphabétique.
   - Si aucune application n'est installée, la liste MesApplis indique Aucun.
   - ********

### [****](#)Installation d'une appli - États du bouton

L'installation de l'application nécessitant le traitement de la requête par le serveur et son transfert vers l'appareil, le bouton « Installer » n'affiche pas la progression en temps réel. Son état passe par : Installer > Demandé > Installé.

**Procédure**

1.  Connectez-vous à  avec votre périphérique iOS. 
2. **** Appuyez sur l'icône Apps@Work. 
3. Appuyez sur **Installer**. Les notifications d'état s'affichent comme suit : 
   - Un message d'alerte s'affiche, la première fois, pour signaler qu'une installation a été demandée.
   - Appuyez sur le bouton demandé. Un message d'alerte s'affiche.

 Le statut « Installé » n'est pas un bouton.

### [****](#)Applications présentées et bannière

L'onglet « À la une » est visible selon la configuration définie par l'administrateur. Il s'agit de la page d'accueil par défaut lorsqu'aucune mise à jour n'est disponible.

 ![](<Resources/Images/Application vedette.png>)

**Procédure**

1.  Connectez-vous à  avec votre périphérique iOS. 
2. **** Appuyez sur l'icône Apps@Work. 
3. Appuyez sur l'onglet **Présenté**. 
   - La bannière Appli présentée affiche une seule application.
   - La section Appli présentée contient la liste de toutes les applis présentées.

     ![](<Resources/Images/Featured Banner.png>)



### [****](#)Notification de mise à jour de l'application

L'utilisateur reçoit une notification sur son appareil lorsqu'une mise à jour d'application est disponible. Cette notification indique le nombre d'applications concernées. En cliquant dessus, l'application Apps\\@Work s'ouvre.

**Procédure**

1. Connectez-vous depuis votre appareil iOS. L'icône Apps\\@Work affiche le nombre d'applications nécessitant une mise à jour.
2. Touchez la notification de mise à jour de l'application ; vous serez redirigé vers l'onglet « Toutes les applications » dans Applications\\@Work. Les indications suivantes s'affichent :
   -  La sous-section Mises à jour disponibles de l'onglet Toutes les applis indique le nombre des mises à jour d'application disponibles.
   - Une icône de point rouge s'affiche pour chaque application nécessitant une mise à jour.

     ![](Resources/Images/Updates.png)



### [****](#)Paramètres - Mes périphériques

**Procédure**

1.  Connectez-vous à  avec votre périphérique iOS. 
2. Appuyez sur l'icône **Paramètres**.
   - L'onglet Mes périphériques est disponible sous Paramètres.
   - Mes périphériques est désormais le nom d'une ligne sous Authentification.
