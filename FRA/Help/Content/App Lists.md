****Vous pouvez créer des listes pour les applis obligatoires, les applis de la liste d'autorisations ou les applis de la liste de blocages à utiliser avec la stratégie Applis autorisées. Ces listes peuvent vous aider à définir des mesures spécifiques à prendre dans le cas où les applis installées sur les périphériques ne respectent pas les exigences imposées par les listes d'applis. Vous ne pouvez pas modifier les listes d'applications une fois créées, car les listes d'applications peuvent être référencées dans [**Politiques des applications autorisées**](\<./Monitoring and Controlling Allowed Apps.md>). De même, vous ne pouvez pas modifier de listes d'applis figurant dans des stratégies Applis autorisées.

## Création de listes d'applis

Procédure

1. Cliquez sur **Admin** **>** **Infrastructure** **>** **Listes d'applis**.
2. Cliquez sur **Créer une liste**.
3. Sous **Nom de liste d'applis**, entrez un nom pour la liste.
4. Choisissez un type de liste : **Liste blanche**, **Liste noire** ou **Requis**.
5. Sous **Ajouter des applis**, sélectionnez un type d'appli, à savoir **App Store**, **Magasin OS X**, **Google Play** ou **Catalogue d'applis**.
6. Saisissez des critères de recherche pour restreindre les résultats.
7. Utilisez les cases à cocher pour sélectionner les applis. Vous pouvez faire plusieurs recherches et cocher plusieurs cases.
8. ********Cliquez sur l'onglet Afficher les applis pour obtenir la liste des applis sélectionnées jusque là.


Vous pouvez maintenant utiliser cette liste lorsque vous configurez la politique [**Applications autorisées**](\<./Monitoring and Controlling Allowed Apps.md>).

Si la page **Liste des applis** ne s'affiche pas, c'est peut-être parce que vous ne disposez pas des permissions requises. Vous avez besoin de l'un des rôles suivants [**rôles**](\<./User Roles.md>) :

- Gestion du système
- Lecture seule du système
