****

À partir de la version 91, le certificat d'identité des appareils Apple est automatiquement renouvelé dans les 30 jours suivant son expiration. Les appareils iOS reçoivent des certificats MDM renouvelés d'Ivanti Neurons for MDM lors de leur enregistrement régulier. Cependant, il est nécessaire de réinscrire les appareils iOS auprès d'Ivanti Neurons for MDM lorsqu'ils restent hors ligne suffisamment longtemps pour que le certificat MDM expire avant un nouvel enregistrement qui aurait permis son renouvellement.

- L'utilisation de SHA-1 est déconseillée lors de la création de certificats d'identité. Vous pouvez choisir d'autres algorithmes. Lors de la mise à jour des certificats, si les anciens certificats utilisent SHA-1, le même algorithme SHA-1 peut être réutilisé. En revanche, si les anciens certificats utilisent un algorithme supérieur à SHA-1, le passage à SHA-1 est impossible.
- Après avoir configuré un certificat d'identité, vous pouvez cliquer sur **Tester la configuration et continuer** pour émettre et vérifier la validité du certificat de test. Une erreur peut s'afficher lors de ce test pour une configuration de certificat d'identité générée dynamiquement, nouvelle ou existante, si le nom du sujet est identique à celui de l'autorité de certification locale. Dans ce cas, vous devez modifier le nom du sujet du certificat d'identité, qui doit être différent de celui de l'autorité de certification locale. Pour les configurations de certificats d'identité existantes dont le nom du sujet a été modifié, les certificats sont réémis et les configurations sont mises à jour.

  Si vous avez configuré l'option permettant de créer une configuration sans envoyer de certificat de test pour la distribution de certificats **générés automatiquement**, cliquez sur **Continuer**.
- Les certificats sont réémis implicitement.
- Lorsque les certificats d'identité sont affectés à des applis Android, l'appli de l'utilisateur obtient des certificats d'identité sans inviter les utilisateurs à octroyer la permission (au lieu de l'appli) à utiliser le certificat. 
- La messagerie peut être configurée avec un certificat d'identité fourni par l'utilisateur, puis déployée et attribuée comme configuration d'application sur les appareils Android d'entreprise. Cette fonctionnalité est uniquement disponible pour les profils professionnels sur les appareils appartenant à l'entreprise et en mode propriétaire de l'appareil.
