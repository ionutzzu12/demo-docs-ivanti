管理者は、Windowsデバイス上でAutopilotプロファイルを作成することができます。 前準備として、Entra IDユーザソースを構成する必要があります。

**手順**

1. 管**理者 > Microsoft Azure > Autopilotに**移動します。
 Entra ID ユーザー ソースが構成されていない場合、「 追**加 **」ボタンは無効になります。 **[Microsoft Azure]** セクションに表示される **[Entra IDユーザソース]** オプションを使用してユーザソースを構成する必要があります。
2. ********[Windows Autopilotプロファイルを追加] ページが画面に表示されます。
3. **[名前]** ボックスにプロファイル名を入力します。
4. 以下のテーブルを使用して **[プロファイル設定]** を完成させます。
5. ****すべてのEntra IDグループが入った新しいページが画面に表示されます。
6. Autopilotプロファイルを割り当てるEntra IDグループを1つ以上選択します。 
7. AutopilotプロファイルをすべてのEntra IDグループに割り当てる場合は、**[すべてのEntra IDグループに割り当てる]** オプションを選択します。 
8. **[完了]** をクリックします。


| 設定 | 説明 |
 | --------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
 | **デバイス タイプ** | デバイスに応じて次の 2 つのオプションのいずれかを選択します。 <br />**** Windows
* **HoloLens** - このオプションを選択する場合は、デフォルトの導入モードを **[自己導入]** モードに設定しておく必要があります。<br /> まれに、Autopilotを使用してHoloLens 2デバイスを登録すると、「仕事用のデバイスのセットアップ」画面で登録が止まることがあります。 そのような場合は、電源ボタンを押してデバイスの電源を一旦オフにし、再度オンにします。 デバイスにログイン画面が表示されますので、ここでEntra ID認証情報を入力して登録を完了させます。********自己導入：このモードでは、ユーザがほとんど、あるいはまったく介入せずにデバイスの導入が行われます。 
- **ユーザ主導**：このモードでは、新しいWindowsデバイスの構成をユーザが制御して使用可能な状態にすることができます。 ********管理者：デバイスの導入後、ユーザが完全に制御する必要がある場合は、このオプションを選択します。
* **標準**：デバイスの導入後、ユーザが基本的なオプションに対する権限を必要とする場合は、このオプションを選択します。****デフォルトでは、言語はオペレーティングシステムによって決まります。 リストから別の言語を選択できます。|
