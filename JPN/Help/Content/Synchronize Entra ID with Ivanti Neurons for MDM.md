Entra IDとIvanti Neurons for MDMを同期させるために、Graph APIの代わりに、System for Cross-domain Identity Management（SCIM）が採用されました。 SCIMでは、管理者は以下の内容を実行できます。 
- ユーザおよびグループの部分的な同期
- すべてのユーザおよびすべてのグループの同期


System for Cross-domain Identity Management（SCIM）は、Ivanti Neurons for MDM アプリケーションと任意のサービスプロバイダーとの間でのユーザのIDデータ交換を安全に自動化するために使用される、クロスドメインのID管理システムです。 SCIMは、マルチプラットフォーム、マルチ環境に対応し、多くのIDプロバイダーに対応しています。 管理者が各種アプリケーションのユーザアカウントを手動で作成したり削除したりする必要はありません。 SCIMは時間を節約するほか、人的ミスを削減し、Ivanti Neurons for MDM 上でのユーザの移動を経済的、迅速、簡単にします。 割り当てられたユーザだけでなく、すべてのユーザおよびグループのプロビジョニングが可能です。 SCIMの機能フラグがオフになっている場合は、SREチケットを作成し、Ivanti Neurons for MDM でプロビジョニングオプションを有効にしてもらうことができます。

さまざまなIDサービスプロバイダー向けにIvanti Neurons for MDMで追加されたSCIMエンドポイントを、現在Entra IDで使用しているエンドポイントで再利用することができます。

## Ivanti Neurons for MDM からのトークンの生成

管理者はSCIM機能を有効化するために、SREチケットを作成する必要があります。 SCIMを利用するには、Ivanti Neurons for MDM からのトークンとURLが必要です。 

**手順**

1. Ivanti Neurons for MDM にログインします。
2. **[管理]** をクリックします。 [カスタム属性] ページが開きます。
3. 左ナビゲーションペインで、**[Microsoft Azure]** セクションまでスクロールし、**[Entra IDユーザプロビジョニング]** をクリックします。[Entra IDユーザプロビジョニング] ページが開きます。
4. 新しいトークンを生成するには **[生成]** をクリックします。新しいページが開き、トークンの詳細とターゲットのSCIM URLが表示されます。
5. **[コピー]** をクリックすると、トークンまたはSCIM URLのいずれかがコピーされます。
6. 既存のトークンの状態を変更するには、[Entra IDユーザプロビジョニング] ページの **[選択]** ドロップダウンボタンをクリックします。
7. [選択] ボタンを使って、トークンに以下のような変更を加えることができます。
   - **アクティブに設定**
   - **非アクティブに設定**
   - **更新**
   - **削除**

8. 一度に生成できるトークンの数は最大2個です。


## Entra IDとIvanti Neurons for MDMの間の接続を確立する

Entra ID企業アプリケーションでユーザとグループを作成した後、Azure Entra IDとIvanti Neurons for MDMの間の接続を確立することができます。

**手順**

1. Azure Entra IDポータルにログインします。
2. **[企業アプリケーション]** を開き、自分のアプリケーションを選択します。
3. **[プロビジョニング]** > **[プロビジョニングの編集]** > **[管理者認証情報]** を開きます。
4. Ivanti Neurons for MDM管理ポータルからターゲットのSCIM URLをコピーして、Azure Entra IDポータルの **[テナントURL]** フィールドに貼り付けます。
5. Ivanti Neurons for MDM管理ポータルからトークンをコピーして、Azure Entra IDポータルの **[シークレットトークン]** フィールドに貼り付けます。
6. **[試験接続]** をクリックします。 緑色のチェックが入ったポップアップが表示され、接続が確認されます。
7. テナントURLは固定です。 トークンだけが変化し続けます。
資格情報を保存するには、[ **保存** ] をクリックします。


## 割り当てられたユーザのプロビジョニング

Azure Entra IDとIvanti Neurons for MDM間の接続が確立した後は、ユーザのプロビジョニングを実行できます。

**手順**

1. Azure Entra IDポータルにログインします。
2. **[ユーザおよびグループ]** から **[+ユーザ/グループを追加]** をクリックします。 [割り当てを追加] ページが開きます。
3. **[検索]** フィールドからユーザを検索し、**[選択]**,をクリックしてから **[割り当て]** をクリックします。 [ユーザおよびグループ] ページが開きます。
4. 割り当てられたユーザに対して、チェックボックスを選択します。
5. **[プロビジョニング]** をクリックし、**[プロビジョニングを開始]** をクリックします。 プロビジョニングが成功するとその詳細が表示され、ユーザが Ivanti Neurons for MDM にプロビジョニングされます。


## 割り当てられたユーザのプロビジョニングの確認

割り当てられたユーザがAzure Entra IDでプロビジョニングされた後、Ivanti Neurons for MDM管理ポータルでプロビジョニングを確認します。

**手順**

1. Ivanti Neurons for MDM 管理者ポータルにログインします。
2. メインメニューで、**[ユーザ]** タブを開きます。 プロビジョニングされたユーザは、このページのユーザのリストに表示されます。


## 割り当てられたグループのプロビジョニング

割り当てられたグループをプロビジョニングすることで、そのグループ内のすべてのユーザをプロビジョニングすることができます。

**手順**

1. Azure Entra IDポータルにログインします。
2. **[ユーザおよびグループ]** から **[+ユーザ/グループを追加]** をクリックします。 [割り当てを追加] ページが開きます。
3. **[検索]** フィールドでグループ名を指定し、グループを選択します。
4. **[選択]** をクリックした後、**[割り当て]** をクリックします。 [ユーザおよびグループ] ページが開きます。
5. **[ユーザおよびグループ]** でリストからプロビジョニングするグループを選択し、**[プロビジョニング]** > **[プロビジョニングを開始]** を選択します。


## グループのプロビジョニングの確認

Entra ID上でグループがプロビジョニングされた後、Ivanti Neurons for MDM上でプロビジョニングを確認します。

**手順**

1. Ivanti Neurons for MDM 管理者ポータルにログインします。
2. **[ユーザ]** タブ> **[ユーザグループ]** を開きます。 プロビジョニングされたグループは、このページのグループのリストに表示されます。

 Entra ID ではプロビジョニング間隔は 40 分に設定されています。
