登録の招待状にはセルフサービスポータルへのリンクが含まれます。 エンドユーザは、このポータルを利用して以下のタスクを実行できます。
- ロック
- ロックの解除
- 位置情報の表示 ([**プライバシー設定**](\<./Privacy Configuration.md>) で有効になっている場合)
- ワイプ
- 撤去
- アカウント情報の変更（名前、パスワード、Eメールアドレス）
- 強制チェックイン
- 暗号化証明書と署名証明書を追加


エンドユーザは、セルフサービスポータルに表示されている登録ポータルリンクをクリックし、追加のデバイスを登録します。

[****](https://mydevices.mobileiron.com/user/login.html)エンドユーザがセルフサービスポータルのURLを紛失した場合は、https://mydevices.mobileiron.com/user/login.htmlをエンドユーザに送信してください。 iOS ユーザーの場合は、セルフサービス ポータル用に [**Web クリップ構成**](\<./Create a Web Clip Configuration.md>) を作成することを検討してください。

## 署名証明書と暗号化証明書のアップロード

セルフサービスポータルの [ユーザ提供の証明書] 構成では、エンドユーザにメールの署名証明書と暗号化証明書のアップロードを許可することができます。 この設定は [ユーザ提供の証明書] 構成で構成します。 構成すると、エンドユーザが署名証明書と暗号化証明書をアップロード可能となります。

1. **[マイ証明書]** タブで **[証明書を追加]** をクリックします。 **[証明書を追加]** ウィンドウが表示されます。
2. 次のフィールドを更新します。 


| フィールド名 | 説明 |

 | --------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |

 | 証明書の種類 | アップロードする証明書の種類を選択します。 選択肢は次のとおりです。 <br />****暗号化証明書
* **署名証明書**<br /> これらのオプションは管理ポータルから作成します。 詳細については、[**Identity 証明書の構成**](\<./Identity Certificate.md>) を参照してください。 ****[ファイルを選択] をクリックし、アップロードする証明書を選択します。 <br /> ファイルがPKCS12形式であることを確認してください。ファイルに使用するパスワードを入力します。|


3. **[アップロード]** をクリックします。


アップロードした後は、以下の詳細を示す表で証明書のリストを確認できます。

| フィールド名 | 説明 |
 | ---------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
 | 証明書名 | 証明書の種類 ( **暗号化** または **署名**のいずれかを指定します。                                                                                                                                                                                    発行された証明書の詳細。証明書がアップロードされた日付。|

 | 有効期限 | 証明書の有効期限。                                                                                                                                                                                                                         以下のアクションを実行できます。 <br />****証明書の編集 - 証明書の詳細の編集。 
* **プライベートキーをクリア** - 証明書パッケージ（PKCS#12）からプライベートキーを削除します。
* ****証明書の削除 - サーバーから証明書を削除します。|


ユーザが証明書の構成をアップロードすると、証明書を使用している構成をサーバーが再プッシュします。

ユーザがプライベートキーを削除またはクリアした場合は、構成が再プッシュされません。

詳細については、[**セルフサービス ポータル ブランディング**](\<./セルフサービス ポータル _ブランディング_.md>) を参照してください。
