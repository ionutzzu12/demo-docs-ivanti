Oktaユーザプロビジョニングでは、SCIMプロトコルを使用してOktaをと同期させ、部分的なユーザとグループの同期を可能にします。 Oktaユーザプロビジョニングでは、SCIMプロトコルを使用して、Oktaからに供給されるユーザオブジェクトとグループオブジェクトを自動的に作成および更新します。  管理者は、ディレクトリサービス全体を同期させるか、特定のユーザオブジェクトおよびグループオブジェクトをと同期させるかを選択できます。 現在のOktaとの統合と同様に、ユーザおよびグループのプロビジョニングプロセスが自動化されます。 Oktaでユーザまたはグループに対して変更が行われると、同じ変更がにも反映されます。 最も重要な違いは、Oktaユーザプロビジョニングでは、特定のユーザおよびグループをプロビジョニングできるという点です。 これにより、管理者は、 で追加、更新、無効化されるユーザとグループを特定するためにより厳密に制御できます。 

このトピックには、現在入手可能なOktaドキュメントと関連性がある情報が記載されています。 最新のドキュメントについては、関連する公式Webサイトをご覧ください。

## ****トークンの生成 

Oktaユーザプロビジョニングを開始するには、でトークンとターゲットURLを生成します。

必ずトークンとターゲット URL を保存してください。

一度に生成できるトークンの数は最大2個です。

**手順**

1.  管理ポータルにログインします。 
2. **[管理] **>** [ID] **>** [ユーザ プロビジョニング]** を開きます。
3. **[ID プロバイダ (IdP) の選択]** ドロップダウンメニューから **[Okta]** を選択します。
4. 新しいトークンを生成するには、**[生成]** をクリックします。 通知メッセージが表示されます。**[生成]** をクリックします。 新しいページが開き、トークンの詳細とターゲットの SCIM URLが表示されます。
5. **[コピー]** をクリックしてトークンとSCIM URLをコピーします。 **次に、Oktaと間の接続の確立トピックのステップを実行します。
6. ページを更新します。 **[ユーザプロビジョニング-Okta]** ページにトークンステータス表が表示されます。


## ポータルからのトークンステータスの変更

既存のトークンの状態を変更できます。

**手順**

1. **[ユーザ プロビジョニング-Okta]** ページで **[選択]** ドロップダウンメニューをクリックします。
2. **[選択]** をクリックし、トークンに対して次の変更を行います。
   - **アクティブに設定**
   - **非アクティブに設定**
   - **更新**
   - **削除**



## ****Oktaと間の接続の確立

Oktaアプリケーションでユーザとグループを作成した後、Oktaと間の接続を確立できます。

**手順**

1. Oktaポータルにログインします。
2. ************[アプリケーション] > [アプリカタログの参照] > [ SCIM 2.0テストアプリ（OAuthベアラトークン）の追加] を開きます。 
3. **[SCIM 2.0テストアプリ（OAuthベアラトークン）]** をクリックします。
4. **[プロビジョニング]** > **[統合]** を開き、**[編集]** をクリックします。
5. ****管理ポータルからターゲットSCIM URLをコピーして、Oktaポータルの [SCIM 2.0ベースURL] フィールドに貼り付けます。
6. ****ポータルからトークンをコピーして、Oktaポータルの [OAuthベアラトークン] フィールドに貼り付けます。
7. **[API認証情報のテスト]** をクリックし、**[保存]** をクリックします。 Oktaととの間で接続が確立されます。


## ****Oktaポータルからのユーザとグループのプロビジョニング

Oktaととの間で接続が確立された後、ユーザやグループをプロビジョニングできます。 

**手順**

1. Oktaポータルにログインします。
2. **[アプリケーション]** > **[プロビジョニング]** を開き、**[編集]** をクリックします。
3. すべてのオプションの **[有効化]** チェックボックスを選択し、**[保存]** をクリックします。
4. **[ディレクトリ]** を開き、**[ピープル]** をクリックします。    1. **[人の追加]** をクリックし、すべての関連フィールドに入力して、**[保存]** をクリックします。
   2. **[グループ]** を開いて **[グループを追加]** をクリックし、関連フィールドに入力して、**[保存]** をクリックします。
   3. グループ名をクリックします。 グループの詳細が開きます。
   4. **[人の割り当て]** をクリックし、**[メンバー]** リストからメンバーを選択し、**[保存して戻る]** をクリックして **[完了]** をクリックします。

5. **[アプリケーション]** > **[アプリケーション]** を開き、**[SCIM 2.0テストアプリ（OAuthベアラトークン）]** をクリックします。   1. **[割り当て]** を開き、**[割り当てる]** ドロップダウンメニューをクリックして、**[人への割り当て]** をクリックします。 [割り当て] ページにユーザがリストされます。
   2. ユーザを選択し、**[保存して戻る]** をクリックして、**[完了]** をクリックします。
   3. **[割り当て]** を開き、**[割り当てる]** ドロップダウンメニューをクリックして、**[グループへの割り当て]** をクリックします。 [割り当て] ページにユーザがリストされます。
   4. グループ名を選択して **[割り当てる]** をクリックし、**[保存して戻る]** をクリックして、**[完了]** をクリックします。グループが割り当てられます。

6. **[グループのプッシュ]** タブを開き、**[グループのプッシュ]** ドロップダウンメニューをクリックして、**[名前によるグループの検索]** をクリックします。
7. **[名前でグループをプッシュ]** フィールドにグループ名を指定し、**[今すぐグループメンバーシップをプッシュ]** チェックボックスを選択します。
8. **[保存]** をクリックします。 ステータスが [アクティブ] と表示されます。 グループがポータルに表示されるようになります。


## ****ポータルでのユーザプロビジョニングの確認

割り当てられたユーザがOktaポータルでプロビジョニングされた後、管理ポータルでプロビジョニングを確認します。

**手順**

1.  管理ポータルにログインします。 
2. **[ユーザ]** を開き、**[ユーザ]** をクリックします。 Oktaポータルでプロビジョニングされたユーザは、**[ユーザ]** ページにリストされ、ソースはSCIM Oktaと示されます。


## ****ポータルでのグループプロビジョニングの確認

グループがOktaポータルでプロビジョニングされた後、でプロビジョニングを確認します。

**手順**

1.  管理ポータルにログインします。 
2. **[ユーザ]** タブを開き、**[ユーザグループ]** をクリックします。 プロビジョニングされたグループは、このページのグループのリストに表示されます。 会員数を確認することができます。

プロビジョニングプロセスには最大 1 時間かかる場合があります。


## ****ポータルでの設定の編集

このトピックでは、Okta設定の構成について説明します。 

**手順**

1. **[管理] **>** [ID] **>** [ユーザ プロビジョニング-Okta]** を開きます。
2. **[トークンの生成]** をクリックして、トークンをコピーします。
3. ページを更新します。 [設定の編集] ページが開きます。
4. **[設定の編集]** をクリックします。
5. ****Oktaからインポートしたユーザを自動的に招待 - Oktaからにインポートしたユーザに自動的に登録招待メールを送信するかどうかを管理します。
6. （任意）**カスタム属性の追加** - デバイス管理に適用したいカスタム ユーザ属性をディレクトリ サービスから指定します。 これにより、各属性は、変数をサポートする [構成] フィールドの${attributeName}によって参照されます。 このオプションを使用するには、すべてのOktaサーバーにカスタム属性を一貫して実装しておく必要があります。 実装に含まれるOktaサーバーがカスタム属性を使用していない場合、カスタム属性に依存する機能が意図通りに機能しないことがあります。
7. Okta設定を変更した後、[**変更**の**保存**] をクリックします。
