**ローカル管理者パスワードソリューション（LAPS）**は、Windowsデバイスのパスワードの安全な管理を自動化します。 LAPSはランダムな、定期的なパスワード変更を強制的に実施し、それらのパスワードをディレクトリ内に安全に保存して、侵害された場合の水平展開のリスクを軽減します。

最新のLAPSがAzureおよび最新のWindowsデバイス管理プラットフォームに統合されており、Ivanti Neurons for MDMとAzureポータルを使用したパスワードの安全な保管、取得、およびポリシー構成が可能になっています。 これにより、レガシーとクラウド管理型の両方のWindowsエンドポイント全体で、管理者の認証情報が確実に保護され、監査可能な状態が保たれます。

****でLAPSを使用するには、Azure管理者はN-MDMアプリが必要なLAPS許可（DeviceLocalCredential.Read.All）とともにAzureテナントに追加されるようにする必要があります。

**** LAPS構成では、構成が適用されるローカル管理者アカウントを、ポリシー設定管理者アカウント名を使用することで指定できます。 構成で指定されているアカウント名がデバイス上に存在しない場合、そのアカウント名は管理対象となりません。 ただし**管理者アカウント名**が空白のままになっている場合、ポリシーは、デフォルトであるデバイス組み込みローカル管理者アカウント（その既知の相対識別子（RID）によって識別される）に設定されます。

LAPS構成が適用されると、デバイスのローカル管理者アカウントのパスワードが管理者によって管理され、パスワードがAzure Active Directoryで生成されるようになります。

### LAPSのその他の機能

-  Azureへのパスワード保存 - ローカル管理者パスワードをAzureデバイスオブジェクトに安全に保存できるようになりました。
- パスワードの取得 – 管理者はデバイスパスワードを直接Azureポータルを通じて取得できます。
- 構成可能な設定 – LAPSの構成可能な設定をポータルで管理できます。
- オンデマンドでのパスワードローテーション – 管理者はポータルからデバイスパスワードを手動でローテーションできます。
- 使用されたパスワードの自動リセット – 一度アクセスされたパスワードを自動的に再設定できます。


### LAPS構成の作成

前提条件 
- デバイス用にLAPSソリューションを有効化する必要があります。 このソリューションはデフォルトでは全員が使用できるわけではありません。 支援については、テクニカルサポートチームまでお問い合わせください。
- デバイスをAzureデバイス運用（ADO）に登録する必要があります。


**手順**

1. 構成管理プラットフォームで、LAPS構成の作成を開始します。
2. 次の**必須パラメータを設定します**。
   - **バックアップ ディレクトリ** - Microsoft Entra IDのみへのパスワードのバックアップを選択します。
   - **ローカル管理者名** - システム上の管理者アカウント名を指定します（例：Admin）。

3. **追加の設定を構成** - 組織のセキュリティポリシーに従い、次の項目を設定します。 指定しない場合はデフォルト値が使用されます。
   - **パスワードの有効日数** - ローカル管理者のパスワードが自動的にローテーションされるまでの日数。
   - **パスワードの複雑さ** - パスワードの複雑さの要件を指定します（例：英数字、特殊文字）。
   - **パスワードの長さ** - ローカル管理者のパスワードとして必要な長さを設定します。
   - **認証後のアクション** - 認証後に実行されるべきアクションを定義します。
   - **認証後のリセット遅延** - 認証後にパスワードをリセットするまでの遅延（時間 / 分）を設定します。

4. **LAPS 構成を配布**
   - このLAPS構成プロファイルを、ADOに登録されたユーザ/デバイスに割り当てます。
   - 設定を適用すると、ランダムなローカル管理者パスワードが作成され、Microsoft Entra ID の次の場所に安全に保存されます。
     **デバイス** → **デバイスの管理** → **ローカル管理者のパスワード回復**。

5. **パスワードのローテーション**
   - パスワードは、**パスワードの有効日数**に設定された値に従って自動的にローテーションされます。
   - パスワードのローテーションのステータスはデバイスログで確認できます。



## パスワードのローテーション

ローカル管理者パスワードの即時ローテーションを実施するには、以下の手順に従ってください。

1. **デバイスに移動** - 管理コンソールでデバイス詳細ページを開きます。
2. **パスワードローテーションを開始**
   - 3点メニュー（…）をクリックします。
   - **管理者のローカル パスワードを循環させる**を選択する。
   - プロンプトが表示されたらアクションを確認します。 これでパスワードローテーション要求が送信されます。

3. **チェックインして確認**
   - デバイスは、次回の強制チェックイン中にパスワード変更を処理します。
   - Microsoft Entra IDでページを更新すると、パスワードリカバリセクションに更新済みのパスワードが表示されます。
