監査証跡のエクスポートとは、すべての監査証跡情報を特定のサーバーロケーションにエクスポートおよびアップロードする機能です。 サーバーはデフォルトポートからアクセスできる必要があります。 ユーザは監査証跡エクスポート設定から、監査証跡のアーカイブを特定のロケーションに毎日自動的にアップロードするよう設定可能です。

 監査証跡のエクスポートは、LinuxベースおよびWindowsベースのSFTPサーバーでのみ可能です。

監査証跡エクスポート設定を構成するには：

1. **[管理] > [インフラストラクチャ] > [監査証跡]** を選択します。 **[監査証跡]** ページが表示されます。
2. **[監査証跡] **ページで **[ON]** をクリックし、監査証跡のエクスポートを有効化します。
3. **[エクスポート]** セクションで、以下のフィールドを更新します。


| **機能** | **説明** |
 | -------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
 | **エクスポート形式** | 監査証跡データをエクスポートする形式を以下から選択します。<br />* **JSON**
* ****CEFログメッセージは以下のデフォルト値を含みます。
  - バージョン：CEFバージョン番号。 現在サポートされているバージョンはv25です。
  - デバイス ベンダ: Ivanti Inc
  - デバイス製品: Ivanti Neurons for MDM
  - デバイスバージョン：イベント生成時点で最新の Ivanti Neurons for MDM バージョン
  - デバイスイベントクラスID：証跡に固有のエンティティID
  - 名前：エンティティ名と証跡ごとのアクション 例：プロモーション配布構成設定の作成
  - 重大度：イベントの重要性を指定。 例：低

* CEFログメッセージには、以下のキーと値のペアを集めた拡張フィールドもあります。 
  - CS1、CS1Label：createdAt、entityType、entityName、actionTypeなどの監査証跡メタデータ
  - CS2、CS2Label：行為者情報
  - CS3、CS3Label：アクション状態の前
  - CS4、CS4Label：アクション状態の後<br />CEFエクスポートでは、いずれかのフィールド（CS3やCS4のキー）が所定の制限値を超えると、実際の値の代わりに「Value for this key exceeds mapped dictionary key allowed length（このキーの値は対応の辞書キーに許可された長さを超えています）」のテキストが入ります。****監査証跡をエクスポートするサーバー名を入力します。****サーバーにログインするユーザ名を入力します。****サーバーログインパスワードを入力します。****サーバーのパスを入力し、サーバーに所定のパスが存在することを確認します。 例：/Users/Test/Export****送信用SFTP構成の監査ログをエクスポートするためのキー交換アルゴリズムを選択します。<br />デフォルトでは次のキー交換アルゴリズムが選択されています。<br />* **ディフィー・ヘルマン・グループ交換 sha1**

* **ディフィー・ヘルマン・グループ14-sha1**
* **curve25519-sha256**
* **diffie-hellman-group-exchange-sha256** (デフォルトで選択) |
 | **暗号** | 送信 SFTP 構成の監査ログをエクスポートするための暗号化暗号のリストを選択します。 デフォルトでは次の暗号化方式が選択されています。<br />* **aes128-ctr**
* **aes192-ctr**
* ********送信用SFTP構成の監査ログをエクスポートするためのHMACアルゴリズムのリストを選択します。<br />hmac-sha1 （デフォルト選択）
* hmac-sha2-256
* hmac-sha2-512


3. 上記に挙げたフィールドは、構成済みの場合、読み取り専用になっています。 構成済みのフィールドを編集するには、**[編集]** ボタンをクリックします。 管理者がすでに SFTP エクスポートを設定している場合は、アップグレード後にすべてのキー アルゴリズムが選択されます。
サーバー接続をテストし、監査証跡のエクスポート構成を保存するには、「 **テスト接続」と「保存」をクリックします。
アーカイブされた監査証跡ファイルは、JS**ON 形式の .zip ファイルで入手できます。
監査証跡のエクスポート設定を保存する前に、すべてのフィールドの構成設定を確認してください。無効な値がフィールドに入力されている場合はエラーメッセージが表示されます。
