手順

1. ******** にログインし、[管理] > [Microsoft Azure]に移動します。 
2. ********左側のメニューで [Microsoft Azure] > [デバイスコンプライアンス] をクリックします。
3. ************[iOS、macOS、Androidのデバイスコンプライアンス] セクションまでスクロールし、必要なテナントタイプを選択します。 [追加] をクリックします。
   - [標準] を選択すると、既存のフローに従い、「.com」URLが使用されます。
   - [GCC High] を選択すると、米連邦政府または米国防総省（DOD）の特定の要求に応じた「.us」URLを通して通信が送信されます。

4. [アカウントを接続] の下で次の詳細情報を入力します。
   - AzureテナントID - Microsoft Azureインスタンスを参照します。
   - 登録URL（任意） - デバイスがMDM登録されていない場合、この登録用URLがデバイスユーザに表示されます。 設定時にはHTTPS形式を使用してください。 組織内でページをホストし、デバイスユーザに登録情報を表示したい場合は、ここにリンクを追加します。 
   - 修復URL（任意） - デバイスがコンプライアンス違反の場合、この修復用URLがデバイスユーザに表示されます。 設定時にはHTTPS形式を使用してください。 組織内でページをホストし、デバイスユーザに修復情報を表示したい場合は、ここにリンクを追加します。

5. [アカウントを接続] をクリックします。 [Azureテナント検証] ダイアログボックスが開きます。


現在Azureに接続されており、macOSデバイスのデバイスコンプライアンスの追加を希望するテナントは、アカウントの接続を切断した後、接続を再確立する必要があります。

1. [Azureテナント検証] ダイアログで、手順1にあるリンクをクリックします。 
2. ログインします。 
3. 許可の内容を読み、[許可する] をクリックします。


ログイン後のページで、再度ログインするよう求められた場合は、そのブラウザタブ/ウィンドウを閉じてください。

1.  に戻ります。 [Azureアカウントの連携] ダイアログボックスで [同意しました] チェックボックスを選択します。 [確認] をクリックします。 <Resources/画像/デバイスコンプライアンス1.png>)


- アカウントを編集するには [アカウントを編集] をクリックします。
- アカウントの接続を解除するには [アカウントを切断] をクリックします。 詳しい説明はAzureテナントのプロビジョニング解除を参照してください。
- アカウントの追加、編集、無効化のすべてのアクティビティはログに記録されます。
