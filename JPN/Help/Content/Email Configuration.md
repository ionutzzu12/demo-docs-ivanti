Eメール構成では、デバイスのPOPまたはIMAP Eメールを設定します。

## Eメールの設定

| **設定** | **対処方法** |
 | ------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
 | 名前 | この構成を識別する名前を入力します。                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           この構成の目的を明示する説明を入力します。このEメールアカウントを識別するために使用するテキストを入力します。|

 | アカウントの種類 | IMAP または POP を選択します。 IMAPを選択した場合、パスの接頭辞も入力できます。 使用可能なアカウントのタイプについては、各インターネットサービスプロバイダ（ISP）までお問い合わせください。 接頭辞は、一般的に、すべてのIMAPフォルダを受信トレイに表示する際に必要となります。 接頭辞を必要とするISPは通常、設定する特定の接頭辞について情報を提供しています。Eメールアカウントを識別するために使用するテキストを入力します。 ユーザはこの値をデバイス上でも設定できるという点に注意してください。アカウントのEメールアドレスを指定する変数を入力します。Eメールがこのアカウントから移動するのを防ぐ必要がない場合に選択します。S/MIME暗号化サポートを有効にする場合に選択します。 次に署名と暗号化証明書を選択します。<br />証明書のキャッシュが必要です。 ID証明書の構成で使用されている認証機関でキャッシュが有効化されていることをご確認ください。<br />**iOS 10.3+：**<br />**[S/MIME署名]** と **[S/MIME暗号化]** フィールドで以下のいずれかを選択します。<br />オフ
* オン
* ユーザ選択<br />iOS 12.0+：<br />ユーザがS/MIME署名設定をオーバーライドできるようにする
* ユーザがS/MIME署名IDを選択できるようにする
* ユーザがS/MIME暗号化設定をオーバーライドできるようにする
* ユーザがS/MIME暗号化IDを選択できるようにする<br />必要に応じて **[S/MIMEメッセージごとの署名と暗号化を有効化]** します。このアカウントでMail Dropを許可する場合に選択します。 Mail Dropによりユーザは、メールに添付するファイルをiCloudに保存し、そのリンクをメールで送信することにより、重いファイルでもメール添付で送信できます。 Mail Drop の詳細については、次を参照してください: [**https://support.apple.com/**](https://support.apple.com/) |
 | アプリごとの VPN | メール ドメインに複数の異なるアプリごとの VPN プロファイルを関連付ける機能は、Apple によってサポートされています。 Per-App VPNを使用したIMAPとPOP3のメール設定もサポートされました。<br />**必須事項**：メール構成でPer-App VPNを設定する前にTunnelまたはPer-App VPNを構成してください。<br />ドロップダウンメニューで **[Per-App VPN構成] を選択します**。 |


### 受信メール

| **設定** | **対処方法** |
 | -------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------- |
 | メール サーバーおよびポート | このアドレスはインターネット サービス プロバイダー (ISP) から提供されます。                                                                         受信メールサーバーにアクセスするためのユーザ名を入力します。 多くの場合、Eメールアドレスと同一となります。 お使いのISPが形式を提供しています。|

 | 認証タイプ | ISP によって定義された認証タイプを選択します。                                                                                     受信メールサーバーにアクセスするためのパスワードを入力します。デバイスとサーバー間の通信には、セキュアソケットレイヤー（SSL）のみを選択し、使用してください。|

### 送信メール

| **設定** | **対処方法** |
 | ---------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
 | メール サーバーおよびポート | このアドレスはインターネット サービス プロバイダー (ISP) から提供されます。                                                                                                                                           送信メールサーバーにアクセスするためのユーザ名を入力します。 多くの場合、Eメールアドレスと同一となります。 お使いのISPが形式を提供しています。|

 | 認証タイプ | ISP によって定義された認証タイプを選択します。                                                                                                                                                       送信メールサーバーにアクセスするためのパスワードを入力します。|

 | 送信パスワードは受信パスワードと同じ | SMTP 認証で POP/IMAP と同じパスワードを使用する場合に選択します。                                                                                                                                        この構成をEメールクライアントのみが使用するようにしたい場合に選択します。 ネイティブEメールクライアントを使用してコンテンツを送信するアプリなど、Eメールを送信するその他のアプリは、この構成を使うことができません。デバイスとサーバー間の通信には、セキュアソケットレイヤー（SSL）のみを選択し、使用してください。|
